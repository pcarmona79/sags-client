Introduction

SAGS or Secure Administrator of Game Servers is an Open Source project who has the finalty to produce a solution to manage game servers under GNU/Linux.

The project consists in a client-server application which uses a SSL connection to transfer game information and necessary commands to manage it.

It has been released under the terms of the GNU General Public License.

The website's project is hosted at Sourceforge.net and the URL is http://sags.sourceforge.net/.

Features

  • The client provides a direct interface to the console of authorized game servers.
  • It have panels to show list of processes, process's information and consoles.
  • You can change the console's font and the program remembers the value.
  • The login dialog stores five entries of servers.
  • You can save the console's messages to a file.
  • Linux version supports IPv6.
  • IRC-based chat support.
  • Chat window manages normal messages, actions (/me), notices (/notice) and private messages from other users. /topic commands. Also shows when users joins and leaves the channel and using the /topic command you can change the channel's topic.
  • It is portable to Windows platform (you need Dev-C++ to compile it and will need OpenSSL libraries to run it) because it has a GUI writed with wxWidgets.

How to get SAGS Client

Go to the project's website and get into "Downloads" section. At end of page you will find the "Precompilated binaries" title next to a link to a installer (for win32) or RPM packages. The downloading will use the  Sourceforge.net's downloading system, so probably you will need to select a mirror before download the file.

The source code of both client and server can be downloaded directly from "Downloads" section or from the File Archive at Sourceforge.net: http://sourceforge.net/project/showfiles.php?group_id=106892.

How to use

The following image shows the main window. It contains three principal elements which are Processes List, Process's information panel and many tabs that shows processes's consoles, chat channel and program's logs.


Connection

The first thing that you must do is to connect to remote server using the username and password asigned to you by the administrator. To enter the data go to Session / Connect... menu. Then a window like the following will be showed.


Enter server's IP like <IP address>:<port> and then your username and password. In Linux version, you can use IPv6 addresses like [<IPv6 address>]:<port>, only if the operative system supports this new protocol.

If you are connecting to a server at 47777 port, it's not necessary to enter this value, only the IP address.

Session

During the session, the main window must be like the following image which shows some Quake II servers at the Processes list.


Selecting a name at Processes list, the console of that process will be shown and the information will be displayed at panel Process's information. Only authorized processes for the user will be showed by the program.

Para enviar un comando al servidor del juego seleccionado, sólo escríbalo en la entrada de abajo y luego pulse el botón Enviar o simplemente presione la tecla Enter. Los comandos ingresados son enviados al servidor y pasados directamente al proceso del juego. En la consola queda destacado el comando enviado al servidor y la salida es mostrada en la consola unos instantes después. El comando enviado es guardado en el menú desplegable de esa entrada para que pueda ser usado posteriormente.

Se puede cambiar la fuente de la consola escogiendo Ver / Preferencias... en el menú y luego presionando el botón correspondiente a Fuente de las consolas.

Para guardar los mensajes de la consola actual a un archivo hay que seleccionar Proceso / Guardar en archivo... en el menú.

En la pestaña Registros se pueden ver las distintas acciones que realiza el cliente y se muestran los nombres de algunos de los paquetes enviados por el servidor. Se pueden guardar los registros en un archivo de texto seleccionando Pestaña / Guardar registros en archivo... en el menú. Se puede ocultar esta pestaña escogiendo Ver / Mostrar registros en el menú. Este estado será recordado por el programa.

Chat

La pestaña Conversación muestra a todos los usuarios conectados al servidor y permite que se puedan comunicar entre ellos. Los usuarios administradores son indicados con una ampolletita verde.

Los mensajes son enviados al resto de los clientes con sólo escribirlo en la entrada inferior y presionando Enter o haciendo clic en el botón Enviar. También están disponibles los siguientes comandos para enviar otros tipos de mensaje:

  • /me: envía un mensaje al canal como una acción.
  • /notice: envía un mensaje al canal como una noticia.
  • /topic: cambia el tópico del canal sólo si el usuario que invoca el comando es administrador.
  • /to nick text: envía text al usuario nick como un mensaje privado (sólo lo recibe nick).

Se puede cambiar la fuente del canal escogiendo Ver / Preferencias... en el menú y luego presionando el botón correspondiente a Fuente del canal de chat.

Para guardar los mensajes del canal de chat se escoge Pestaña / Guardar conversación en archivo... del menú.

Finishing

To get disconnected, select Session / Disconnect in the menu or just close the application from its title bar's icon.
